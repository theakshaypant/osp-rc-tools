#### Root cause analysis

**4 distinct failure modes across the release PRs:**

1. **Missing Dockerfiles / source code (operator index 4.17, 4.22, 4.23, 5.0; tektoncd-results retention-policy-agent)**
   - Component configs reference Dockerfiles that don't exist on `release-v1.15.x`
   - Operator: `index-5.0.Dockerfile` never created; `index-4.17` has empty `dockerfileUrl`; 4.17 is out of support; 4.22, 4.23, 5.0 likely don't support 1.15
   - tektoncd-results: `retention-policy-agent.Dockerfile` exists on `main` but not `release-v1.15.x`. The source code (`cmd/retention-policy-agent`) also doesn't exist on the release branch — this component was added in a newer version. The hack repo config should not include it for 1.15
   - **Resolution:** Remove these components from the 1.15 Konflux config in the hack repo

2. **`groupadd: command not found` (p12n-opc, tektoncd-results)**
   - Dockerfiles use `groupadd`/`useradd` on `ubi8/ubi-minimal` which doesn't include `shadow-utils`
   - **Fix:** PRs [#289](https://github.com/openshift-pipelines/p12n-opc/pull/289) and [#1224](https://github.com/openshift-pipelines/tektoncd-results/pull/1224) — **both CI PASSED**

3. **Invalid RPM lockfile (tektoncd-results)**
   - `rpms.in.yaml` requests `shadow-utils` but lockfile was never resolved — all `packages: []` empty
   - `hermeto 0.55.0` now enforces stricter validation, rejecting empty package arrays
   - **Fix:** PR [#1224](https://github.com/openshift-pipelines/tektoncd-results/pull/1224) — cleared lockfile to `arches: []` — **CI PASSED**

4. **Missing Yarn tarball + corepack mismatch (p12n-console-plugin-pf5)**
   - Dockerfile runs `npm install -g yarn-1.22.22.tgz` but the file doesn't exist in the repo
   - Builds are hermetic (no network access), so yarn must be vendored
   - Upstream `package.json` has `"packageManager": "yarn@4.6.0"` which makes Yarn 1.22.22 refuse to run via corepack check
   - **Fix:** PR [#168](https://github.com/openshift-pipelines/p12n-console-plugin-pf5/pull/168) — vendor tgz in `.konflux/`, strip `packageManager` field — **CI IN PROGRESS**
