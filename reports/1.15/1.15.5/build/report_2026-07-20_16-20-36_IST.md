#### Root cause analysis

**4 distinct failure modes across the release PRs:**

1. **Missing Dockerfiles / source code (operator indices; tektoncd-results retention-policy-agent)**
   - Component configs reference Dockerfiles that don't exist on `release-v1.15.x`
   - Operator: `index-5.0.Dockerfile` never created; `index-4.17` has empty `dockerfileUrl`; 4.17 is out of support; 4.22, 4.23, 5.0 don't support 1.15
   - tektoncd-results: `retention-policy-agent.Dockerfile` exists on `main` but not `release-v1.15.x`. Source code (`cmd/retention-policy-agent`) also doesn't exist on the release branch — component added in a newer version
   - **Resolution:** Remove these components from the 1.15 Konflux config in the hack repo

2. **`groupadd: command not found` (p12n-opc)**
   - Dockerfile uses `groupadd`/`useradd` on `ubi8/ubi-minimal` which doesn't include `shadow-utils`
   - Hermetic builds have no network access — `microdnf install` only works if packages are in the RPM prefetch
   - Resolved with `rpm-lockfile-prototype --image ubi8/ubi-minimal:latest` which auto-resolves transitive deps (libsemanage)
   - **Fix:** PR [#289](https://github.com/openshift-pipelines/p12n-opc/pull/289) — **CI PASSED**

3. **Invalid RPM lockfile (tektoncd-results)**
   - `rpms.in.yaml` requests `shadow-utils` but lockfile was never resolved — all `packages: []` empty
   - `hermeto 0.55.0` now enforces stricter validation, rejecting empty package arrays
   - Dockerfiles use `ubi8/ubi` (full UBI, NOT minimal) which already includes `shadow-utils` — no RPM prefetch needed
   - **Fix:** PR [#1224](https://github.com/openshift-pipelines/tektoncd-results/pull/1224) — clear packages from `rpms.in.yaml`, set `arches: []` in lockfile

4. **Pre-existing broken build (p12n-console-plugin-pf5)**
   - Component `console-plugin-pf5-1-15-console-plugin` has **never built successfully** on `release-v1.15.x` — no snapshots exist (only `next` and `nightly` variants have snapshots)
   - Multiple cascading issues: missing `yarn-1.22.22.tgz`, corepack `packageManager` mismatch (upstream migrated to Yarn 4.x), stale `.konflux/yarn.lock`, and `yarn install --offline` with no populated cache
   - The npm prefetch (`.konflux/npm/package.json`) only fetches `@yarnpkg/cli-dist: 4.6.0` (the yarn tool), not project dependencies — hermetic builds have no way to install npm deps offline
   - **Resolution:** Needs someone familiar with the console-plugin Konflux build setup to properly configure npm prefetch for project dependencies. Fix PR [#168](https://github.com/openshift-pipelines/p12n-console-plugin-pf5/pull/168) closed — cannot be fixed without proper prefetch setup
   - **Impact on release:** If console-plugin-pf5 has never built on 1.15, it may be shipping the last successful image from a previous build. Verify whether the existing image in the core snapshot is acceptable for 1.15.5
